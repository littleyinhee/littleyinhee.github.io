矩阵是不太好处理的，所以考虑从一行去推下一行。

设上一行选择了 p1,p2,⋯ ,pkp_1,p_2,\cdots,p_kp1​,p2​,⋯,pk​ 这几个横坐标的位置为 111，分情况讨论一下这一行选择的 xxx 位置。

（下列结合自己画图理解）

  • x≥p1x\ge p_1x≥p1​。

首先发现如果选择一个 x∈px\not\in px​∈p，则一定不满足条件。

然后也可以发现，设这一行的选择的 x≥p1x\ge p_1x≥p1​ 的部分为 qqq，则 qqq 是 ppp 的一个前缀。

但是又发现，如果有一行完全不选，则这一行状态可以视为继承上一行的。这个要特判一下。

  • x<p1x<p_1x<p1​。

结合前面，发现可以任选。

于是我们可以开始 dp 了。

设 dpi,j,kdp_{i,j,k}dpi,j,k​ 表示枚举到第 iii 行，这一行选了 jjj 个 111，最左侧的 111 左边还有 kkk 个 000 的方案数。

首先转移 x≥p1x\ge p_1x≥p1​ 的情况。枚举转移到长度为 lll 的前缀，直接转移即可。

然后在上一步的基础上转移 x<p1x<p_1x<p1​ 的。枚举选 aaa 个，最左边一个左边有 lll 个 000，另用一个数组转移。最后全部加到一起。

于是能写出如下 O(n5)O(n^5)O(n5) 的代码，n,mn,mn,m 同阶：

int n,m,dp[2][N][N],f[N][N],s[N];
il int Mod(int x,int y){return x+y>=mod?x+y-mod:x+y;}
void solve3(){
	dp[0][0][m]=1;
	rep(i,1,n){
		int p=i&1;
		mems(f,0),mems(dp[p],0);
		rep(k,0,m){
			s[0]=dp[p^1][0][k];
			rep(j,1,m)s[j]=Mod(s[j-1],dp[p^1][j][k]);
			rep(l,0,m)f[l][k]=Mod(f[l][k],Mod(s[m],mod-s[l-1]));
		}
		rep(j,0,m){
			rep(k,0,m){
				if(j)dp[p][j][k]=Mod(dp[p][j][k],dp[p^1][j][k]),dp[p][0][k]=Mod(dp[p][0][k],mod-dp[p^1][j][k]);
			}
		}
		rep(j,0,m){
			s[0]=0;
			rep(k,1,m)s[k]=Mod(s[k-1],f[j][k]);
			drep(l,m,1)f[j+1][l-1]=Mod(f[j+1][l-1],Mod(s[m],mod-s[l-1]));
		}
		rep(j,0,m)rep(k,0,m)dp[p][j][k]=Mod(dp[p][j][k],f[j][k]);
	}
	int ans=0;
	rep(i,0,m)rep(j,0,m)ans=Mod(ans,dp[n&1][i][j]);
	printf("%d\n",ans);
}
