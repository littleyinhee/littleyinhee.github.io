Sol 1

首先有个一定需要的 O(n2)O(n^2)O(n2) dp：设 dpu,idp_{u,i}dpu,i​ 表示当前在 uuu 点，钦定选的链上点权一定不包含 iii，此时的最小代价。转移是非常简单的，只要看当前链是否在这里断开即可。而且 mex⁡\operatorname{mex}mex 性质是很好的，如果此时转移的 iii 并不是 mex⁡\operatorname{mex}mex，也一定不会成为最优解，而是真正的 mex⁡\operatorname{mex}mex 会成为最优解。于是这么做是正确的。

然后会观察出一个问题：我们很多时候并不会取到很大的 mex⁡\operatorname{mex}mex。然而，我们 dp 的第二位就是和这个上界相关。而根据证明（参考 cf 上的 edtorial），这个上界是 O(nln⁡n)O(\dfrac{n}{\ln n})O(lnnn​) 的！我们也可以通过一个构造证明：12131421516321，即把每个数和它的因数降序列出。感性理解一下发现很有道理。你考虑分成 nnn 组。对于每个数 xxx，你都会希望每 xxx 长度就有一个 xxx，否则消耗了这么多的长度会亏。并且通过提交证明这是对的。

于是就限制第二维枚举到大概 400040004000，就可以 O(n2ln⁡n)O(\dfrac{n^2}{\ln n})O(lnnn2​) 解决了。

code：

bool Mbe;
int n,m,c[N],dp[N][M],ls[N],rs[N];
void dfs(int u){
    if(!u){
        return;
    }
    dfs(ls[u]),dfs(rs[u]);
    if(!ls[u]&&!rs[u]){
        rep(i,1,m){
            if(i!=c[u]){
                dp[u][i]=0;
            }
        }
        if(u==1){
            puts("0");
        }
        return;
    }
    if(!rs[u]){
        int mn=inf;
        rep(i,1,m){
            if(i!=c[u]){
                dp[u][i]=dp[ls[u]][i];
                mn=min(mn,dp[u][i]+i);
            }
        }
        rep(i,1,m){
            if(i!=c[u]){
                dp[u][i]=min(dp[u][i],mn);
            }
        }
        if(u==1){
            printf("%d\n",mn);
        }
        return;
    }
    int x=inf,y=inf,mn=inf;
    rep(i,1,m){
        if(i!=c[u]){
            x=min(x,dp[ls[u]][i]+i);
            y=min(y,dp[rs[u]][i]+i);
        }
    }
    rep(i,1,m){
        if(i==c[u]){
            continue;
        }
        mn=min(mn,dp[ls[u]][i]+dp[rs[u]][i]+i);
        dp[u][i]=min({dp[u][i],dp[ls[u]][i]+y,dp[rs[u]][i]+x});
    }
    rep(i,1,m){
        if(i!=c[u]){
            dp[u][i]=min(dp[u][i],mn);
        }
    }
    if(u==1){
        printf("%d\n",min(mn,x+y));
    }
}

Sol 2

先有 O(n2)O(n^2)O(n2) dp，然后观察到一个性质：对于所有没在以 uuu 为根的子树内出现过的点权 xxx，dpu,xdp_{u,x}dpu,x​ 都是相等的！原因显然。而且我们 dp 的转移是非常好维护的，于是可以使用线段树合并或启发式合并解决。时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn) 或 O(nlog⁡2n)O(n\log^2 n)O(nlog2n)。

一开始是出来作 D 的，想的是一个完全暴力的指数级做法。结果被 co 一眼秒了 O(n2)O(n^2)O(n2) 做法：

于是我们决定 F 就放这个 O(n2)O(n^2)O(n2) 的 dp 问题。但此时发生了两件事：一是，我们发现 O(n3)O(n^3)O(n3) dp，也就是存链的底端是什么，然后计算链上 mex⁡\operatorname{mex}mex 转移的暴力 dp，我们卡不掉。二是 co 问我们，是否能构造出最优情况下选择的链中 mex⁡\operatorname{mex}mex 尽可能大的数据。

经过一番思考，设 mex⁡=m\operatorname{mex}=mmex=m，我们很快得到了一个 mm=nm\sqrt m=nmm​=n 的构造：121321432154321...。但是过了几天，zlt 说他有了一种更强的构造：12131421516321，即把每个数和它的因数降序列出。感性理解一下发现很有道理。你考虑分成 nnn 组。对于每个数 xxx，你都会希望每 xxx 长度就有一个 xxx，否则消耗了这么多的长度会亏。并且通过提交证明这是对的。

然后 zlt 再通过更严谨的证明得出确实就有 mln⁡m=nm\ln m=nmlnm=n，也就大概有 m=nln⁡nm=\dfrac{n}{\ln n}m=lnnn​。于是，就有了这个奇怪的范围和这道有点毒瘤的题。
